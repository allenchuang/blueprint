## Overview

The **Blueprint Marketing Dashboard** (`apps/admin`) is a standalone Next.js app that gives you a real-time view of your social media presence, an AI-powered content composer, and a live feed of what's trending in tech and AI.

It ships inside the Blueprint monorepo and runs on its own port (default `3002`).

### Pages at a glance

| Page | Route | What it does |
|---|---|---|
| Overview | `/` | Follower count, engagement metrics, charts, recent tweets |
| Social Channels | `/socials` | Per-platform analytics, tweet list, post composer |
| Compose | `/compose` | AI-assisted tweet composer with Skylar suggestions |
| Hot Trends | `/trends` | Live trending topics from Hacker News & Reddit |

## Connecting Twitter

The dashboard pulls live data from the Twitter v2 API. You need four credentials from a [Twitter Developer App](https://developer.twitter.com/en/portal/dashboard) with **read + write** permissions.

### 1. Create a Twitter Developer App

1. Go to [developer.twitter.com](https://developer.twitter.com) and create a project + app.
2. Under **User authentication settings**, enable **OAuth 1.0a** with read/write permissions.
3. Generate your Access Token and Secret (for your own account).

### 2. Set environment variables

Add to `apps/admin/.env.local`:

```bash
TWITTER_API_KEY=your_api_key
TWITTER_API_SECRET=your_api_secret
TWITTER_ACCESS_TOKEN=your_access_token
TWITTER_ACCESS_SECRET=your_access_token_secret
```

### 3. Restart the admin app

```bash
pm2 restart admin
# or locally:
cd apps/admin && pnpm dev
```

The dashboard will now show real data. Without credentials, every section displays a clean empty state with a **Connect Twitter →** CTA — no crashes, no broken UI.

### What data is fetched

| API call | Data |
|---|---|
| `GET /api/twitter` | Profile: username, followers, following, tweet count |
| `GET /api/twitter/tweets` | Last 10 tweets with engagement metrics |
| `GET /api/twitter/analytics?period=7d` | Daily impressions, likes, retweets, replies for last 7 days |

All API routes live in `apps/admin/src/app/api/twitter/`.

## Compose Page

The Compose page (`/compose`) lets you write and post tweets without leaving Blueprint. It has two modes:

### Basic Composer

A full-featured tweet editor with:
- Character counter (280-char limit enforced)
- Thread support — add/remove tweet cards
- Preview before posting
- **Post Tweet** button that hits `POST /api/twitter/post`

### AI Suggestions via Skylar

Below the composer, Skylar surfaces a batch of ready-to-post tweet ideas based on your recent content and current trends. Click any suggestion to load it into the composer. Hit **Refresh** to cycle through a new batch (suggestions rotate to avoid repetition).

Suggestions are generated by `GET /api/skylar/suggestions`, which calls your configured LLM with context about your profile.

### Compose from a Trend

On the **Hot Trends** page, each trend card has a **✍️ Write about this** button. Clicking it opens the Compose page pre-loaded with an AI-drafted tweet about that trend — ready to edit and post.

## Hot Trends Page

The Trends page (`/trends`) surfaces real-time topics from two sources:

| Source | What's pulled |
|---|---|
| **Hacker News** | Top stories from `news.ycombinator.com/topstories` |
| **Reddit** | Hot posts from tech/AI subreddits (r/programming, r/MachineLearning, etc.) |

### Relevance scoring

Each trend is scored **High / Medium / Low** relevance:

- **High** — Hacker News stories, or Reddit posts with >500 engagement
- **Medium** — Reddit posts with 100–500 engagement  
- **Low** — Everything else

### Skylar's daily brief integration

At the top of the Trends page, a **☀️ Daily Brief** banner shows Skylar's morning summary — an AI-written digest of today's top trends tailored to your industry. The brief is generated via `GET /api/skylar/brief` and cached for the day.

### Filters

- Toggle between **All** / **Hacker News** / **Reddit** source filters
- Sort by **Relevance** or **Volume**
- Auto-refreshes every 15 minutes

## Skylar's Daily Brief System

Skylar is Blueprint's built-in marketing AI agent. Every day it:

1. Fetches trending topics from Hacker News + Reddit
2. Pulls your recent tweet performance
3. Generates a personalized brief: what to post about, which trends to ride, content gaps to fill

### API routes

| Route | Description |
|---|---|
| `GET /api/skylar/brief` | Today's brief — top trends, suggested angles, content ideas |
| `GET /api/skylar/suggestions` | Fresh batch of ready-to-post tweet suggestions |

Both routes accept no required parameters — they use your connected Twitter credentials and the live trend feed automatically.

### Customizing Skylar

Skylar's prompt can be tuned in `apps/admin/src/app/api/skylar/brief/route.ts`. The system prompt includes:

- Your Twitter profile context (follower count, recent engagement rate)
- Today's top 10 trends
- Instructions to produce actionable, on-brand content ideas

## Onboarding Flow

New users see a **Welcome to Blueprint Marketing** wizard on their first visit. It walks through:

1. **Welcome** — feature overview (analytics, composer, trends, brief)
2. **Connect Twitter** — instructions + env var reference
3. **Tour** — quick links to each section of the dashboard
4. **Done** — redirects to the main dashboard

Completion is tracked in `localStorage` under the key `blueprint_marketing_onboarded`. To reset the wizard (e.g. for testing), run this in the browser console:

```javascript
localStorage.removeItem('blueprint_marketing_onboarded');
location.reload();
```

## Running locally

```bash
cd apps/admin
pnpm dev   # starts on http://localhost:3002
```

Or from monorepo root:

```bash
pnpm dev --filter=admin
```

## Deploying

```bash
NODE_OPTIONS="--max-old-space-size=4096" pnpm build --filter=admin
pm2 restart admin
```

The app is served by PM2 on port `3002` in production.
