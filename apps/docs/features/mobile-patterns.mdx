## Overview

On mobile viewports (< 768px), `apps/web` uses native-app patterns instead of desktop web patterns. This includes a fixed top navigation bar, sticky bottom tab bar, bottom sheets, and slide-in panels. These components are built into Blueprint and ready to use.

## Core components

### `MobileTopNav`

Fixed top bar with a back chevron (left), centered title, and optional right action slot. Use on any screen that can be navigated back from.

```typescript
import { MobileTopNav } from "@/components/layout/mobile-top-nav";

<MobileTopNav title="Settings" rightSlot={<Button>Save</Button>} />
```

### `MobileFooter`

Sticky bottom tab bar for primary navigation. Includes safe-area handling for iOS home indicator.

```typescript
import { MobileFooter } from "@/components/layout/mobile-footer";

<MobileFooter />
```

### `MobileLayout`

Wrapper that renders `MobileTopNav` + `MobileFooter` on mobile, and passes children through unchanged on desktop. Use this as the layout for pages that need mobile-native chrome.

```typescript
import { MobileLayout } from "@/components/layout/mobile-layout";

export default function DashboardPage() {
  return (
    <MobileLayout title="Dashboard">
      <YourContent />
    </MobileLayout>
  );
}
```

### `BottomSheet`

Drag-to-dismiss bottom drawer built on shadcn `Drawer` (powered by [vaul](https://vaul.emilkowal.ski/)). Use for contextual actions, filters, and confirmations.

```typescript
import { BottomSheet } from "@/components/ui/bottom-sheet";

<BottomSheet trigger={<Button>Open</Button>}>
  <p>Sheet content</p>
</BottomSheet>
```

### `SlideInSheet`

Full-screen slide-from-right panel with its own back navigation header. Use for detail views that should feel like a native push navigation.

```typescript
import { SlideInSheet } from "@/components/ui/slide-in-sheet";

<SlideInSheet title="Post Details" trigger={<Button>View</Button>}>
  <PostDetail />
</SlideInSheet>
```

## Responsive detection

### CSS (preferred)

Use Tailwind responsive prefixes to show/hide elements by breakpoint:

```typescript
<div className="md:hidden">Mobile only</div>
<div className="hidden md:block">Desktop only</div>
```

### JavaScript

Use `useIsMobile()` from `@/hooks/use-mobile` when you need JS-level detection:

```typescript
import { useIsMobile } from "@/hooks/use-mobile";

const isMobile = useIsMobile();
if (isMobile) { /* ... */ }
```

## Animations

Animation presets live in `@/lib/mobile-animations`. These are Framer Motion variants tuned with spring physics for a native feel:

```typescript
import { pageTransition, tapFeedback, springConfig } from "@/lib/mobile-animations";

<motion.div {...pageTransition}>
  <Content />
</motion.div>
```

## PWA support

Blueprint ships with PWA support for `apps/web`. Generated by `pnpm sync-config`:

| File | Description |
|---|---|
| `public/manifest.json` | Web app manifest |
| `public/icon-192.png` | PWA icon |
| `public/icon-512.png` | PWA icon (large) |
| `public/apple-touch-icon.png` | iOS home screen icon |

Apple Web App meta tags are in `apps/web/src/app/layout.tsx` for iOS "Add to Home Screen" support.

## Rules

- **DO** use `MobileLayout` for pages that need mobile chrome
- **DO** use `BottomSheet` instead of modals for contextual actions on mobile
- **DO** use `useIsMobile()` only when CSS breakpoints are insufficient
- **DO NOT** use `SlideInSheet` for non-navigation content — it implies forward navigation
