## Clone the Repository

```bash
cd /home/deploy
git clone git@github.com:your-org/your-repo.git
cd your-repo
```

## Install Dependencies

```bash
nvm use           # Picks up .nvmrc (Node 23)
pnpm install
```

## Configure Environment Variables

Copy the example files and fill in values:

```bash
# Database (used by packages/db and apps that import @repo/db)
cp packages/db/.env.example packages/db/.env

# Server
cp apps/server/.env.example apps/server/.env
```

At minimum, set:

```bash
# packages/db/.env
DATABASE_URL="postgresql://user:password@your-neon-host.neon.tech/dbname?sslmode=require"
```

<Warning>
Never commit `.env` files. They are `.gitignore`-d by default. If the file contains secrets, keep it on the VPS only.
</Warning>

## Push Database Schema

If this is a fresh database:

```bash
pnpm db:push
```

## Build All Apps

```bash
pnpm build
```

This runs Turbo to build all apps in dependency order. Expect 2вҖ“5 minutes depending on server specs.

## Create PM2 Ecosystem File

Create `ecosystem.config.cjs` at the repo root:

```javascript
module.exports = {
  apps: [
    {
      name: "web",
      cwd: "./apps/web",
      script: "node_modules/next/dist/bin/next",
      args: "start -p 3000",
      env: { NODE_ENV: "production" },
    },
    {
      name: "server",
      cwd: "./apps/server",
      script: "dist/index.js",
      env: { NODE_ENV: "production" },
    },
    {
      name: "admin",
      cwd: "./apps/admin",
      script: "node_modules/next/dist/bin/next",
      args: "start -p 3002",
      env: { NODE_ENV: "production" },
    },
    {
      name: "docs",
      cwd: "./apps/docs",
      script: "node_modules/mintlify/dist/local-preview/index.js",
      args: "--port 3003",
      env: { NODE_ENV: "production" },
    },
    {
      name: "os",
      cwd: "./apps/os",
      script: "node_modules/next/dist/bin/next",
      args: "start -p 7777",
      env: { NODE_ENV: "production" },
    },
  ],
};
```

<Note>
**Why `node_modules/next/dist/bin/next` instead of `npx next`?** PM2 needs a direct path to the binary. The `.bin` shim doesn't work reliably with PM2's process management. Using the resolved path ensures consistent startup.
</Note>

## Start All Apps

```bash
pm2 start ecosystem.config.cjs
```

## Configure Auto-Start on Reboot

```bash
pm2 save
pm2 startup
```

PM2 prints a command to run with `sudo`. Execute it:

```bash
# Example output вҖ” run the actual command PM2 gives you:
sudo env PATH=$PATH:/home/deploy/.nvm/versions/node/v23.x.x/bin pm2 startup systemd -u deploy --hp /home/deploy
```

## Common PM2 Commands

| Command | Purpose |
|---------|---------|
| `pm2 status` | Show all processes |
| `pm2 logs` | Stream all logs |
| `pm2 logs web` | Stream logs for a specific app |
| `pm2 restart web` | Restart a single app |
| `pm2 restart all` | Restart everything |
| `pm2 reload all` | Zero-downtime reload |
| `pm2 stop web` | Stop a single app |
| `pm2 delete web` | Remove from PM2 |
| `pm2 monit` | Real-time monitoring dashboard |
