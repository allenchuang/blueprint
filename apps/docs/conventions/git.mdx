## Commit message format

Blueprint uses [Conventional Commits](https://www.conventionalcommits.org/):

```
type(scope): description
```

### Types

| Type | When to use |
|---|---|
| `feat` | New feature |
| `fix` | Bug fix |
| `chore` | Maintenance (deps, config, tooling) |
| `docs` | Documentation only |
| `refactor` | Code restructuring with no behavior change |
| `test` | Adding or updating tests |
| `style` | Formatting, whitespace changes |

### Scopes

Use the app or package name: `web`, `admin`, `server`, `react-native`, `remotion`, `docs`, `db`. Omit scope for cross-cutting changes.

### Examples

```bash
feat(web): add user profile page
fix(server): handle null email in user creation
chore(db): add posts table schema
docs: update API reference for user endpoints
refactor(web): extract auth logic into use-auth hook
feat(react-native): add voice agent screen
chore: update turbo.json pipeline
```

## Pre-commit checklist

Before committing:

```bash
pnpm lint          # no lint errors
pnpm check-types   # no TypeScript errors
```

## Branching

| Branch | Purpose |
|---|---|
| `main` | Production-ready code |
| `feat/[description]` | Feature branches |
| `fix/[description]` | Bug fix branches |

## What to commit

| Include | Exclude |
|---|---|
| All source files | `.env` files (only `.env.example`) |
| `pnpm-lock.yaml` changes | `node_modules/` |
| Schema migration files | `.next/`, `dist/`, build artifacts |
| Config changes | Generated files (unless intentional) |

## What NOT to commit

- **Never commit `.env`** — it contains secrets. Only commit `.env.example` with placeholder values.
- **Never commit `node_modules/`**, `.next/`, `dist/`, or other build artifacts — these are in `.gitignore`
- **Never commit generated theme files** — `apps/*/src/app/theme.css` is generated by `pnpm sync-config`

## GitHub workflow

```bash
# Create a feature branch
git checkout -b feat/user-profiles

# Make changes, then commit
git add .
git commit -m "feat(web): add user profile page"

# Push and open a PR
git push -u origin feat/user-profiles
gh pr create --title "feat(web): add user profile page"
```
