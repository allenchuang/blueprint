## Prerequisites

- **Node.js ≥ 23** (run `nvm use` — an `.nvmrc` is included)
- **pnpm 10+** — `npm install -g pnpm`
- A [Neon PostgreSQL](https://neon.tech) database (free tier works fine)

## Option A — Scaffold a new project

Use the Blueprint CLI to create a new project and pick only the features you need:

```bash
npx blueprint new my-app
```

The interactive prompt lets you select apps (web, admin, mobile, docs, remotion) and optional integrations (auth, payments, voice, analytics). Unselected features are removed from the scaffold.

## Option B — Clone directly

```bash
git clone https://github.com/your-org/blueprint my-app
cd my-app
pnpm install
```

## Set up environment variables

```bash
cp .env.example .env
```

Open `.env` and set your database URL:

```bash
DATABASE_URL="postgresql://user:password@host/dbname?sslmode=require"
```

All other variables are optional — features degrade gracefully when their env vars are missing. See [Get Started](/get-started) for the full variable reference.

## Initialize the database

```bash
pnpm db:push
```

This pushes the Drizzle schema to your Neon database. No migrations needed for initial setup.

## Start all apps

```bash
pnpm dev
```

| App | URL |
|---|---|
| Web | [http://localhost:3000](http://localhost:3000) |
| API Server | [http://localhost:3001](http://localhost:3001) |
| Admin | [http://localhost:3002](http://localhost:3002) |
| Docs | [http://localhost:3003](http://localhost:3003) |
| Remotion | [http://localhost:3004](http://localhost:3004) |

## Running individual apps

```bash
pnpm dev:web      # Web app only
pnpm dev:admin    # Admin panel only
pnpm dev:server   # API server only
```
